# Inputs — Raw Materials from People

Raw research and documents contributed by stakeholders, workers, and contributors. These are **advisory sources** — the owner makes all decisions based on this input.

## Suggested subfolders

| Folder | Who | What's inside |
|--------|-----|---------------|
| `po/` | Product owner | Requirements, dialogs, design references |
| `workerN/` | AI workers | Research, dev docs, per-feature deep dives |
| `owner/` | You | Personal notes, fixes, observations |

## Trust rules

- Never treat inputs as authoritative. Cross-check against actual code.
- Mark contradictions in the task/feature file where the source is consumed.
- Mark source explicitly: `(from PO — advisory)`, `(from worker research — verify)`.

Customise the subfolders to match who actually feeds you material.
