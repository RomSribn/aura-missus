# OKRs

Time-boxed objectives mirrored from a board into the knowledgebase so they are
tracked in-project alongside features and tasks.

## Structure

```
okrs/
  OKRS-SUMMARY.md                     live index — all objectives, grades, status
  RULES.md                            naming + file format + how grades stay in sync
  cycle-N-YYYY-MM-<name>/             one folder per cycle (≈ a month on the board)
    P-O-N-<slug>/                     one folder per objective
      P-O-N.md                        the objective: fields, KRs, progress, links
```

## How it connects

```
Board  →  OKRS-SUMMARY.md  →  per-objective file  →  links to PF / PT / PD / PI artifacts
```

The board is the source of truth for grades/status. The knowledgebase copy adds
the **why**, the **key results**, and **links to the actual work** that moves
each objective.
