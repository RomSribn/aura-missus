# Features

What the system does or should do. Technical or product — doesn't matter. A
feature describes the "what", tasks describe the "do".

A feature can exist without a task (just a spec for later). A task can exist
without a feature (quick fix). Usually both exist together.

**Features must be kept up to date.** When implementation changes, the feature
spec is edited to reflect current state. A feature file always describes how
things work *now*, not how they were planned.
