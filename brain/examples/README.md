# Examples

A small, coherent, worked set showing the conventions in practice. All four
artifacts are about the same imaginary feature — **contact lists** — so you can
see how they cross-link along the flow:

```
PI-0001 (research)  →  PD-0001 (decision)  →  PF-0001 (feature)  →  PT-0001 (task)
```

- `PI-0001-contact-list-storage-options/` — an investigation (folder + report).
- `PD-0001-soft-delete-contact-lists.md` — a decision (flat file).
- `PF-0001-contact-lists/` — a feature spec (folder + main file).
- `PT-0001-contact-lists-crud/` — a task with the full step-file lifecycle.

These use prefix `P` (from `PROJECT`). In a real project, swap `P` for your own.
Delete this `examples/` folder once your real artifacts exist.
