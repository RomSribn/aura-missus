# Roadmaps

Planning snapshots (from a product owner, stakeholder, or board) translated into
our internal artifact map.

Use roadmaps for time-bound external/product planning inputs. Concrete
implementation work still belongs in `tasks/`, product scope in `features/`, and
research in `investigations/`.

## Format

- `PR-NNNN-short-name.md`
- One roadmap snapshot per file.
- Keep the original source path and ingestion context near the top.
